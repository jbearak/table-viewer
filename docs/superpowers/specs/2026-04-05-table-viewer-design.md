# Table Viewer â€” VS Code Extension Design Spec

A read-only Excel file viewer for VS Code that supports `.xlsx` and `.xls` files with live reload, persistent UI state, merged cells, font formatting, and theme integration.

## Goals

- View Excel files generated by R and Stata scripts without editing capabilities
- Remember column widths, row heights, scroll position, active sheet, and tab orientation across reopens
- Render merged cells, bold/italic text, and line breaks faithfully
- Always word-wrap cell text regardless of Excel's wrap setting
- Live-reload when the file changes on disk
- Respect the active VS Code theme
- Toggle number formatting on/off (bold/italic and merges always render)
- GPL-3.0 licensed

## Non-Goals

- Editing or saving changes to Excel files
- CSV/TSV support (future addition, not in this spec)
- Virtual scrolling or pagination (files are under 1,000 rows)
- Cell background colors, font colors, or other style properties beyond bold/italic

## Extension Registration

Custom readonly editor registered in `package.json`:

```json
"customEditors": [
  {
    "viewType": "tableViewer.editor",
    "displayName": "Table Viewer",
    "selector": [
      { "filenamePattern": "*.xlsx" },
      { "filenamePattern": "*.XLSX" },
      { "filenamePattern": "*.xls" },
      { "filenamePattern": "*.XLS" }
    ],
    "priority": "default"
  }
]
```

Implements `vscode.CustomReadonlyEditorProvider`. No editing capabilities.

## Parsing

Two parser adapters normalize to a shared data model:

- **`parse-xlsx.ts`** â€” uses ExcelJS for `.xlsx` files. Full access to bold/italic, number formats, and merged cell ranges.
- **`parse-xls.ts`** â€” uses SheetJS (community edition, Apache-2.0) for `.xls` files. Style fidelity may be lower than ExcelJS but covers the legacy format.

The extension host selects the parser based on file extension, then sends the unified `WorkbookData` to the webview.

## Common Data Model

```ts
interface WorkbookData {
  sheets: SheetData[];
}

interface SheetData {
  name: string;
  rows: CellData[][];       // 2D array, sparse for merged regions
  merges: MergeRange[];      // list of merged cell ranges
  columnCount: number;
  rowCount: number;
}

interface CellData {
  raw: string | number | boolean | null;  // underlying value
  formatted: string;                       // number-format applied
  bold: boolean;
  italic: boolean;
}

interface MergeRange {
  startRow: number;
  startCol: number;
  endRow: number;
  endCol: number;
}
```

Cells inside a merged range (but not the top-left anchor) are omitted from the `rows` array. The renderer uses `merges` to apply `rowspan`/`colspan` on the anchor cell's `<td>`.

## Webview Layout

Top-to-bottom:

### Toolbar

Thin bar at the top with:

- **Formatting toggle** â€” controls whether numbers show their Excel format (e.g., `$1,234.56`, `0.00%`, `MM/DD/YYYY`) or raw underlying values. Bold/italic and merged cells always render regardless of this toggle.
- **Tab orientation toggle** â€” switches sheet tabs between horizontal (top) and vertical (left sidebar).

Button styling matches Sight's toggle pattern (`.toggle` / `.toggle.active` using VS Code button CSS variables).

### Sheet Tabs

Horizontal bar below the toolbar by default, listing all worksheet names. Active tab is highlighted. Clicking a tab switches the displayed sheet.

When vertical mode is active (via toggle or persisted preference), tabs move to a left sidebar.

Default orientation is controlled by the `tableViewer.tabOrientation` VS Code setting. Per-file overrides are persisted in `globalState` and take precedence.

### Table

Plain HTML `<table>` inside a scrollable `<div>` with overflow in both axes:

- **Sticky first row** â€” `position: sticky; top: 0` so the first row (conventionally headers) remains visible while scrolling vertically
- **Merged cells** â€” `rowspan`/`colspan` on the anchor `<td>`
- **Bold cells** â€” `font-weight: bold`
- **Italic cells** â€” `font-style: italic`
- **Line breaks** â€” preserved via `white-space: pre-wrap`
- **Word wrap** â€” always on via `word-break: break-word` on all cells, regardless of Excel's wrap setting
- **Column resizing** â€” drag handles on column headers
- **Row resizing** â€” drag handles on row edges

## Theme Integration

Same approach as Sight:

- Read VS Code CSS variables via `getComputedStyle(document.documentElement)`:
  - `--vscode-foreground`
  - `--vscode-editor-foreground`
  - `--vscode-editor-background`
  - `--vscode-editorGroupHeader-tabsBackground`
  - `--vscode-panel-border`
  - `--vscode-list-activeSelectionBackground`
  - `--vscode-list-activeSelectionForeground`
  - `--vscode-list-hoverBackground`
  - `--vscode-focusBorder`
  - `--vscode-editor-font-family`
  - `--vscode-button-background` (for active toggle buttons)
- `MutationObserver` on `document.documentElement` and `document.body` for style/class changes triggers theme recalculation

All CSS uses these variables â€” no hardcoded colors.

## State Persistence

All persisted state lives in VS Code's `globalState`, keyed by file path.

```ts
interface PerFileState {
  columnWidths?: Record<string, Record<number, number>>;   // sheet name â†’ column index â†’ width
  rowHeights?: Record<string, Record<number, number>>;     // sheet name â†’ row index â†’ height
  scrollPosition?: Record<string, { top: number; left: number }>;  // sheet name â†’ position
  activeSheet?: string;
  tabOrientation?: "horizontal" | "vertical" | null;       // null = use VS Code setting
}
```

- Column widths and row heights saved on resize end, debounced (150ms)
- Scroll position saved on scroll, debounced
- Active sheet saved on tab switch
- Tab orientation override saved on toggle
- LRU eviction when exceeding max stored entries (10,000) to prevent unbounded growth
- State is preserved across live reloads

## Live Reload

- `vscode.workspace.createFileSystemWatcher` watches the open file's path
- On change: re-parse with the appropriate adapter, send new `WorkbookData` to webview via `postMessage`
- Webview replaces data but preserves all persisted state (scroll position, column widths, row heights, active sheet, tab orientation)
- If the previously active sheet no longer exists after reload, fall back to the first sheet and clear saved state for the removed sheet
